# 如何添加新行程

足迹页面由 `src/data/travel.json` 驱动。只需在这个文件里加一行数据，地图和列表会自动更新，无需改任何代码。

---

## 添加一条新行程

打开 `src/data/travel.json`，在 `places` 数组里新增一个对象：

```json
{ "country": "泰国", "region": null, "city": "曼谷", "lat": 13.7563, "lng": 100.5018, "date": "2026-05", "type": "travel", "note": null }
```

保存后，运行 `npm run dev` 在本地预览，确认地图上出现新的点、列表里出现新的条目。

---

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `country` | string | 国家名，中文。中国大陆用 `"中国"`，香港用 `"中国香港"`，澳门用 `"中国澳门"` |
| `region` | string \| null | 中国大陆填省/市/自治区简称（如 `"浙江"`、`"上海"`、`"重庆"`），其他国家填 `null` |
| `city` | string | 城市名 |
| `lat` | number | 纬度（小数，南纬为负） |
| `lng` | number | 经度（小数，西经为负） |
| `date` | string \| null | 格式 `"YYYY-MM"`，如 `"2026-05"`。永久根据地（如老家）填 `null` |
| `type` | string | `"travel"` 为普通行程，`"base"` 为根据地（长期居住地） |
| `note` | string \| null | 可选备注，如 `"武功山"`、`"在读"`，不需要就填 `null` |

### type 字段的视觉区别

- **`"travel"`**：地图上显示青绿色小点（`#64ffda`），出现在按年份分组的时间线里
- **`"base"`**：地图上显示橙色大点（`#f59e0b`），出现在「根据地」区块，格式特殊

---

## 如何查经纬度

推荐方法：在 Google Maps 上找到目标城市，**右键单击**，弹出菜单第一行就是经纬度（格式为 `纬度, 经度`）。

注意：`travel.json` 里的顺序是 `lat`（纬度）在前，`lng`（经度）在后。

---

## date 字段格式

统一使用 `"YYYY-MM"` 格式，例如 `"2026-05"`。

不需要精确到具体日期，月份精度即可。

---

## 中国大陆的 region 字段

`region` 值需要能匹配省份数据里的名称。规则：省份 GeoJSON 里的名称以「省」「市」「自治区」结尾，如 `"浙江省"`，而 `region` 只需要填前缀部分，如 `"浙江"`，代码会自动匹配。

常用值对照：

| region 值 | 对应省份名 |
|-----------|----------|
| `浙江` | 浙江省 |
| `江苏` | 江苏省 |
| `上海` | 上海市 |
| `重庆` | 重庆市 |
| `北京` | 北京市 |
| `云南` | 云南省 |
| `四川` | 四川省 |
| `广东` | 广东省 |
| `内蒙古` | 内蒙古自治区 |

## 验证

```bash
npm run dev
```

打开 http://localhost:4321/travel，确认：
1. 地图上出现了新的点（位置、颜色正确）
2. 下方时间线里出现了新条目（年份分组、格式正确）
3. 中国大陆点：对应省份应变为浅绿色填充

---

## 原则

**只改数据，不改代码**。`travel.json` 是唯一需要维护的文件。地图渲染、列表展示全部自动根据数据生成。
